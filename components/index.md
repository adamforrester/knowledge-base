# Component Briefs

A flat catalogue of per-component field-truth briefs. Each brief is a single component's study ‚Äî what the leading systems converge on, where they diverge, and where the practice lands by default. Briefs are the upstream artefact a real engagement uses to populate its per-component docs page (see 29-per-component-documentation-template) and to seed its component data file (see 30-generated-from-data-documentation ¬ß15 schemas). The catalogue is the practice's opinionated default; the engagement's reality overrides where it has to.

---

## How this catalogue works

**Platform scope.** The catalogue is web-primary with mobile as a reference dimension. The web is the authoritative API surface ‚Äî properties, accessibility, and implementation are written against HTML/ARIA/React; mobile systems (Apple HIG, Material 3 Compose, Fluent) are cited only where a component genuinely diverges on touch or native behaviour. This is a deliberate commitment: most agency DS work is web-first and the `.ai.json` seed comes off the web API surface. A standing native-API section or a platform-agnostic abstraction is a future decision, not the current shape.

**Brief shape.** Every brief follows a 15-section spine: framing, anatomy, properties/API, states and variants, usage guidance, accessibility, content guidelines, motion, internationalization, naming, implementation notes, related and alternative components, field POV evolution, sources cited, and an embedded agent-consumable schema. Sections scale to component complexity ‚Äî a brief on `Button` will be longer than a brief on `Divider`, and that is correct. Voice and depth match the numbered files: prose-led, opinionated, citations to real systems with version dates.

**Research workflow.** Briefs use the existing `_research/` dual-agent workspace with a purpose-built prompt template at `_research/_component-prompt-template.md`. Dual-agent is the default for any component where the field is contested; claude-only is allowed for components where the field genuinely converges (Divider, Spinner, simple primitives). The choice is recorded by the contents of the research folder ‚Äî presence of `external-agent.md` means dual-agent, absence means claude-only.

**Synthesis path.** Default is direct from raw outputs into `components/<slug>.md`. The brief itself is the synthesis. An intermediate `_research/_synthesis/` file is used only when dual-agent outputs diverge enough to need explicit reconciliation.

**Promotion.** When a brief lands: update its entry in this index from "not yet briefed" to a link with one-line hook and status flag; add inline cross-references from the numbered files where the component is now relevant (03 always; 14 / 17 / 28 / 16 / 18 / 13 / 04 / 25 / 30 conditionally); leave the `_inbound/` folder in place as audit record.

**The ¬ß15 schema.** Section 15 of every brief is a fenced YAML block ‚Äî the practice's opinionated default `.ai.json` shape for the component. The schema is a structured projection of the prose, not a parallel artefact; if section 3 says `tone: neutral | critical | success`, the schema says the same thing in machine-consumable form. The first six briefs used free-form YAML so the shape could stabilise through real use; it has now been **formalised** ‚Äî see `_schema.md` for the locked key order, the ALWAYS/SCALES rule, and the `inherits:` / `notes.unverified` conventions, to which every brief conforms. Keys and ordering are locked; values stay free-form prose.

**Intake.** Engagement need pulls a brief forward. The seed target list below covers ~40 components across the seven categories. The list is the discipline; the order is the flexibility. Components without briefs are listed with a `scope:` note describing what the brief will cover.

## The catalogue

### Foundations

- **[Icon](icon.md)** ‚Äî a primitive with two load-bearing decisions, meaningful-vs-decorative and how the set is delivered: the decorative-by-default accessible-name matrix (`aria-hidden` unless meaningfully standalone), the icon-only-button trap (the Icon is never the interactive node ‚Äî the wrapping Button is), the geometric/optical anatomy, the `currentColor` model, the delivery trade (inline-SVG default; legacy icon fonts dead; variable WOFF2 a legitimate modern path for multi-axis sets), and the set-as-versioned-API deprecation/codemod problem. *stable* ‚Äî promoted from claude-only to a **dual-agent** run; the second pass reopened the delivery question the solo brief had flattened.
- **[Avatar](avatar.md)** ‚Äî the person/entity identity surface and presence-dot host: the fallback chain (image ‚Üí initials ‚Üí icon) and its SSR/cached-image flash trap (native `<img onError>` + the `img.complete` check over the JS virtual-`Image()` machine), the contextual accessible-name contract (informative bare avatar = the name; **decorative when adjacent to a visible name** ‚Äî the inverse of Icon's decorative-by-default; the fallback preserves the name), the AvatarGroup face-pile (the overlap + "+N more" overflow + the `role=group`/`aria-hidden`-children group-name-and-count, paralleling Tag/Badge), initials + the deterministic color-from-name (the bitwise hash mod an accessibility-vetted palette + the CJK/RTL name-order problem + color-not-sole-carrier + contrast), shape-as-entity-semantics (circle=person / square=org / hexagon=AI) + sizing, and the presence-dot `overflow:hidden` clipping trap resolved from the host side. *stable* ‚Äî the thirty-fourth brief; stands on Icon (fallback) + Skeleton (loading), hosts Badge's PresenceBadge, hosted by Tag/Combobox tokens + list/comment/nav.
- **[Badge](badge.md)** ‚Äî the small non-interactive indicator, resolved by its genre split (count overlay / presence dot / status pill) and the naming morass it produces (Fluent/Primer's tripartite split vs MUI/Ant's one-Badge-overlay-plus-Chip vs Polaris/Chakra/Spectrum's status-pill `Badge`): the dividing line is the **accessibility contract** ‚Äî the overlay genres are `aria-hidden` with their meaning in the **host's** accessible name ("Inbox, 3 unread"; the floating-"3" trap), the status pill announces its own text ‚Äî which is why the practice adopts the **tripartite split** (`Badge`/`CounterBadge`/`PresenceBadge`); plus the dynamic-count `role=status` live region (from Toast), the wrapper-vs-sibling host model + circular-vs-rectangular overlap math + the cutout stroke + the RTL logical-property flip, color-not-sole-carrier severity (from Banner/Toast), count handling (max-cap/zero), and the hard **Badge-vs-Tag boundary** (non-interactive indicator vs interactive/removable token). *stable* ‚Äî the thirty-second brief; stands on Icon, inherits the Feedback severity + live-region patterns, overlaid on Icon button/Avatar/Tabs/nav hosts.
- **[Tag](tag.md)** ‚Äî the interactive/removable token and the active counterpart to Badge: the genre split (Material's assist/filter/input/suggestion four chips + Carbon's read-only/dismissible/selectable/operational matrix) collapsing to **three interaction archetypes** (action ‚Üí Button, selection ‚Üí Checkbox/Radio, input/token ‚Üí the multi-select Combobox), the naming morass (Chip vs Tag vs Token; Fluent's `Tag`/`InteractionTag` split) resolving to `Tag` + `TagGroup`, the **Badge-vs-Tag boundary from Tag's side** (interactive/removable ‚áí Tag), and the accessibility crux ‚Äî the **removable-tag-group GRID pattern** (`role=grid`/`row`/`gridcell` + Delete/Backspace + the focus-management-after-removal contract, never drop to `<body>`; *why* listbox is prohibited ‚Äî ARIA forbids interactive elements in `role=option`), the selection roles (listbox/radiogroup) + action=button, the embedded-`√ó` target-size trap (2.5.8) + the `display:contents` grid fix, the token-field composition with the Combobox (+ Backspace-removes-last), and truncation/overflow ("+N more"). *stable* ‚Äî the thirty-third brief; stands on Icon + Button, composed by the multi-select Combobox, borders Checkbox/Radio, inherits the Badge/Banner severity pattern.
- **[Divider](divider.md)** ‚Äî the catalogue's thinnest component and sharpest accessibility test: the decorative-vs-semantic separator decision (`aria-hidden` by default, `role="separator"` only for true thematic breaks), the vertical-divider height trap, the inset model, the labelled "OR" divider, and the divider-versus-spacing judgment. *stable* ‚Äî the catalogue's first claude-only brief; proves depth scales to the component (states and motion sections collapse, by design).
- **[Spinner](spinner.md)** ‚Äî the minimal indeterminate inline busy indicator, and one corner of the three-loading-patterns triad (Progress / Spinner / Skeleton): the **no-value contract** (the Progress-vs-Spinner boundary ‚Äî no `aria-valuenow`), the **separate-primitive-vs-`CircularProgress`-variant** decision (Material 3 fractured it out as a distinct "Loading indicator" in May 2025), the accessibility announcement (`role=status` + "Loading" vs indeterminate `progressbar` vs `aria-busy` on the region + the **never-unlabeled** rule), the **button-loading composition** (the `disabled`-vs-`aria-disabled` focus-to-`<body>` trap + width-preservation), the anti-flash + reduce-motion substitution inherited from Progress, the **`disableShrink`** CPU-jank fix, and the **RTL force-clockwise** rule. *stable* ‚Äî the thirtieth brief; inherits Progress's anti-flash/reduce-motion substrate, composed by Button's loading state. Skeleton would complete the loading triad.
- **[Skeleton](skeleton.md)** ‚Äî the content-shaped placeholder for an initial load, and the brief that **closes the three-loading-patterns triad** (Progress / Spinner / Skeleton): the **layout-reserving/CLS contract** (the Spinner opposite ‚Äî match the final dimensions) + the **shape-mismatch trap** (a mismatched skeleton is *worse* than none), the three-loading-patterns decision, the **reduce-motion drop-to-static** (the Spinner difference ‚Äî the cue is the shape, not the motion), the **`aria-hidden`-boxes + `aria-busy`-region + `role=status` dual-node** accessibility (never read the boxes ‚Äî Roselli), the variant model + the **SSR/RSC auto-derive collapse** (auto-derive ‚Üí 0√ó0 server box ‚Üí CLS, so explicit-dimensions/hand-built are the only CLS-safe path), the anti-flash, and the shimmer-vs-pulse-vs-static animation. *stable* ‚Äî the thirty-first brief; inherits Progress/Spinner's anti-flash + `aria-busy` substrate; the loading terminal state of the data-container machine Empty State described. **The three-loading-patterns triad is complete.**

### Layout

### Form

- **[Button](button.md)** ‚Äî the action trigger, not navigation: the two-axis intent + appearance model over a single overloaded variant enum, the link-vs-button boundary, native-disabled vs. focusable-inactive, and the delayed-spinner / width-preserving loading contract. *stable* ‚Äî the catalogue's calibration brief.
- **[Text Field](text-field.md)** ‚Äî the data-capture primitive where accessibility, validation, and composition collide: bundled-props vs. composed-slots resolving to a hybrid, the graduated typed-family split (NumberField separate; email/url/tel as type+attrs), placeholder-is-not-a-label with static top labels, read-only ‚âÝ disabled, the aria-describedby chain and the polite character counter, and presentational-by-default validation. *stable* ‚Äî the second brief; calibrates the encapsulation-boundary and accessibility-wiring axis.
- **[Textarea](textarea.md)** ‚Äî Text Field's multi-line sibling, where ~80% is the shared form-field substrate and the contested 20% is the sizing model: distinct component over a multiline prop, `resize` vertical-by-default (never horizontal) with auto-grow for composers, `rows` as line-count, soft over hard `maxLength` counted by grapheme, the threshold-based polite counter, Enter-as-newline with submit-on-Enter as an opt-in, and `field-sizing: content` retiring the ghost-sizer. *stable* ‚Äî the third brief; calibrates the sizing model and the inherited-substrate discipline (the first brief to stand explicitly on another).
- **[Select](select.md)** ‚Äî the form component where accessibility ideals collide hardest with styling ambitions, and the first whose core is contested at the *platform* level: the native-vs-custom decision (a framework, not a winner), the select-vs-combobox boundary (type-ahead vs. destructive filtering), the multi-select token/count models with mandatory option checkboxes, the `aria-activedescendant` focus contract (DOM focus stays on the trigger), and the Popover API / CSS Anchor Positioning / `appearance: base-select` platform shift that is collapsing the whole native-vs-custom trade. *stable* ‚Äî the fifth brief; calibrates the native-vs-custom decision and writing durably across a moving platform.
- **[Combobox](combobox.md)** ‚Äî the hardest widget in ARIA, and the first brief to stand on two substrates at once (the text-field form-stack and select's overlay/listbox machinery): the defining destructive filter, the two-state model (`inputValue` vs. `value`) as the core reconciliation problem, `aria-autocomplete=list` default, strict-vs-free-text, async debounce + `AbortController` race handling, the multi-select token-vs-checkbox split, and the ARIA 1.2 pattern (role on the input, not the deprecated 1.0 wrapper). *stable* ‚Äî the seventh brief; calibrates the two-state reconciliation and standing on two inherited substrates.
- **[Checkbox](checkbox.md)** ‚Äî the catalogue's decomposition calibration: the same control at three granularities (atomic box, labelled row, group) resolved to a **two-component surface** ‚Äî `Checkbox` + `CheckboxGroup` ‚Äî plus an exposed `Checkbox.Control` primitive (the text-field hybrid applied to a control); the indeterminate-as-DOM-property mechanic, the whole-row hit target, `role=group` over `fieldset`, the Tab-each-vs-arrow-within difference from radio, and the staged-vs-immediate checkbox-vs-switch boundary. *stable* ‚Äî the eighth brief; establishes the selection-control decomposition that Radio (group mandatory) and Switch (no group) refine.
- **[Radio](radio.md)** ‚Äî the Checkbox decomposition with one mutation: the group is **mandatory** (a lone radio is meaningless; `RadioGroup` owns `name` + single value + validation). Confirms the inherited control/group/`Radio.Control` model, resolves the roving-tabindex single-tab-stop keyboard model and the contested selection-follows-focus question (practice: follows-focus, native-aligned, with explicit-selection as a documented opt-out), and covers no-deselect + the explicit "None", start-empty default selection, and the radio-vs-segmented-control / radio-vs-select boundaries. *stable* ‚Äî the ninth brief; refines the selection-control decomposition (mandatory group).
- **[Switch](switch.md)** ‚Äî the control that closes the selection-control decomposition arc: **no group** (checkbox optional, radio mandatory, switch none). The immediate-effect contract (the staged-vs-immediate boundary from the switch side), the async/optimistic-update model (optimistic default + a first-class pending/loading lock), `role=switch` distinct from checkbox and `aria-pressed`, the label-leading settings-row pattern, supported read-only, and describe-the-setting (not the state) labels. *stable* ‚Äî the tenth brief; completes the boolean/choice batch.
- **[Toggle Button](toggle-button.md)** ‚Äî a Button that carries a persistent binary pressed state (`role=button` + `aria-pressed`): the setting-vs-action/mode boundary with Switch, the persistent-`selected`-vs-transient-`:active` trap, the constant-accessible-name + `aria-pressed` pattern, Space-and-Enter activation, and the atomic-vs-group relationship that feeds Segmented Control. *stable* ‚Äî the eleventh brief; claude-only, standing on the Button substrate.
- **[Segmented Control](segmented-control.md)** ‚Äî the group of the Toggle Button as a connected control, and the catalogue's most semantically contested component: the four-model ARIA fork resolved by purpose (`role=group` + `aria-current` for the immediate view-switcher; `radiogroup` only for a deferred form input; `aria-pressed` for multi-select), the two visual paradigms (track-and-pill vs. connected buttons), the segmented-vs-tabs / -radio / -select boundaries, the 2‚Äì5-segment no-overflow rule, `disallowEmptySelection`, and the sliding-indicator implementation. *stable* ‚Äî the twelfth brief; closes the toggle-group pair and refines radio's a11y-model claim.
- **[Slider](slider.md)** ‚Äî the analog fader forced into a discrete DOM, and the last standalone form control: one overloaded component (scalar|tuple value), the step/tick + float-rounding math, the value-display and finger-occlusion problem, the Arrow/Page/Home-End + `aria-valuetext` contract, the slider-vs-NumberField precision boundary resolved via the bi-directionally-synced paired-input composite, and the pointer-capture / thumb-overhang / thumb-collision traps. *stable* ‚Äî the thirteenth brief; completes the standalone form-control set.
- **Date Picker** ‚Äî *not yet briefed.* `scope: input vs. calendar, single vs. range, locale and RTL, the masked-input contract, accessibility patterns.`
- **File Upload** ‚Äî *not yet briefed.* `scope: drag-and-drop vs. browse, multi-file, progress and error handling, accessibility, the file-list slot model.`
- **Form (composite)** ‚Äî *not yet briefed.* `scope: the form as compositional unit, validation strategies, error summary pattern, submit-and-recovery flow, autofill contract.`

### Navigation

- **[Link](link.md)** ‚Äî the navigation primitive, and the routing/`aria-current` side of the boundary Tabs, Segmented Control, and Button all point at: the `<a href>` contract (the no-href / onClick-div anti-pattern), link-vs-button from the link side (Enter not Space), the inline/standalone/quiet variants and the underline discipline (colour-alone fails 1.4.1), internal-vs-external + the new-tab/tabnabbing rules, `aria-current` for nav, and the framework-router polymorphism (asChild/Slot, always a real `<a href>`). *stable* ‚Äî the sixteenth brief; closes the routing thread.
- **[Tabs](tabs.md)** ‚Äî the first Navigation brief and the component most misused for routing: in-page panel switching (the tabs-vs-routing boundary ‚Äî route changes are nav links + `aria-current`, not a tablist; and the tabs-vs-segmented boundary, panels vs. value-in-place), the tablist/tab/tabpanel + roving-tabindex contract, manual-vs-automatic activation (automatic default, manual when switching is expensive), the panel-mounting trilemma, and overflow handling. *stable* ‚Äî the fourteenth brief; opens the Navigation category.
- **[Accordion](accordion.md)** ‚Äî Tabs' vertical, multi-open cousin: progressive disclosure of stacked sections via the heading + `button[aria-expanded]` + region pattern (each header its own tab stop ‚Äî NOT tabs' roving), the Disclosure-primitive decomposition (single collapsible + coordinated set), multi-open default, the native-`<details>`-first decision (the `name` attribute, `::details-content`, `calc-size`), and the long-notorious height-animation problem now solved natively. *stable* ‚Äî the fifteenth brief; the tabs‚Üíaccordion responsive transform ties the two.
- **[Breadcrumbs](breadcrumbs.md)** ‚Äî the first Navigation brief built directly on Link: a trail of ancestor Links inside the APG `nav[aria-label]`+`ol`/`li` landmark, ending at the current page. The settled location-vs-path ontology, the contested current crumb (practice: non-interactive text + `aria-current="page"`, not `location`; the React Aria/Primer interactive-link form flagged), the decorative-separator rule (CSS `::before`, never DOM content or a list item), the `maxItems`/`itemsBeforeCollapse`/`itemsAfterCollapse` overflow-collapse model keeping root + current, the mobile back-up-one-level swap, and the SEO `BreadcrumbList` JSON-LD as an app/SSR concern (the component never emits it silently). *stable* ‚Äî the seventeenth brief; closes the wayfinding-hierarchy lane against Tabs (peers) and Stepper (linear).
- **[Pagination](pagination.md)** ‚Äî the most reuse-heavy brief in the catalogue: it inherits Link + the Breadcrumbs `nav[aria-label]`+list landmark contract (page items + `aria-current`), Button (prev/next/load-more), and Select (rows-per-page), and invents almost nothing. The three-model decision (numbered for bounded/jumpable/bookmarkable sets, load-more as the safe feed default, infinite scroll restricted to casual discovery with a load-more fallback), the truncation/window algorithm (`siblingCount`/`boundaryCount` + the inert `aria-hidden` ellipsis), the link-vs-button page-item rule (real `?page=N` `<a href>` vs SPA Button; `rel=prev/next` deprecated by Google 2019, so crawlable links are the only SEO lever), the under-specified focus-on-page-change problem (live-region "Page N of M" for numbered SPA; **move focus to the first new item** for load-more), the enterprise-table-pager vs simple-content-pager archetypes, and the `tabular-nums` no-layout-shift discipline. *stable* ‚Äî the nineteenth brief; WAI-ARIA APG has no pagination pattern (it *is* nav + links + `aria-current`), which is exactly why it inherits rather than invents.
- **[Side Navigation](side-navigation.md)** ‚Äî the most composition-heavy brief and the one that **completes the Navigation category**: primary persistent wayfinding that inherits Link (items + `nav[aria-label]` landmark + `aria-current`), Drawer (the mobile/collapsed-overlay form), Accordion/Disclosure (sub-sections), and Tooltip (rail labels). The persistent-structure-vs-drawer decision + the responsive transform (a layout column ‚Üí a modal drawer behind a hamburger; Material's rail/drawer as responsive variants of one structure), the collapsed-rail-vs-expanded model (+ the icon-only label requirement, sub-items pivoting to flyout Popovers), the multi-level **Disclosure-not-`role=menu`** sub-nav (the nav-is-not-a-menu correction reaching nav; Ant's `role=menu`-for-nav the flagged anti-pattern) + the Disclosure-Navigation-Hybrid + the side-nav-vs-Tree boundary, the `aria-current` active-state contract, the tabbed-not-roving keyboard model, the **SSR/FOUC cookie-not-localStorage** state persistence, and the **width-animation layout-thrashing** trap. *stable* ‚Äî the twenty-fourth brief; with Tabs, Accordion, Breadcrumbs, Link, Pagination, and Side Navigation, the Navigation category is complete.

### Feedback

- **[Toast](toast.md)** ‚Äî the brief that **opens the Feedback category**: the non-interrupting, transient, no-focus-steal counterpart to Dialog, establishing the ARIA live-region announcement pattern the rest of the category reuses. The no-focus-steal contract, the live-region announcement (`role=status`/polite vs `role=alert`/assertive + the **pre-existing-region NVDA trap**), the auto-dismiss-vs-WCAG-2.2.1 timing tension (pause on hover/focus/blur, ‚â•5s minimums, error/actionable shouldn't auto-dismiss), the action-in-toast keyboard-reachability problem (Radix F8 / React Aria focusable landmark / Carbon's focus-moving ActionableNotification / Primer's ban / Apple's OS-level prohibition; the Radix `aria-live=off`+`altText` visual-vs-announcement split), the queue/stack/limit toaster (Sonner's stacked/expand-on-hover + the `::after` hover-bridge + the portal tab-order focus-proxy fix), severity + color-not-sole-carrier, the **no-error-toasts** POV, and swipe + the button alternative. *stable* ‚Äî the twenty-fifth brief; the first Feedback brief, the non-interrupting opposite of Dialog.
- **[Banner / Alert](banner.md)** ‚Äî the second Feedback brief: the persistent, in-flow, actionable counterpart to Toast, inheriting the live-region pattern Toast established and **refining it with the static-vs-dynamic rule** (a banner present at page load is read in document order and is *not* `role=alert` ‚Äî that's an anti-pattern; only a dynamically-inserted one is a live region). The persistent/in-flow contract (the Toast opposite), the placement taxonomy (page/global vs contextual/section vs the inline-message boundary), the severity + color-not-sole-carrier model, the **dynamic heading-level mapping** trap, the legitimacy of actions (no Toast reachability problem in-flow), dismissibility + don't-re-nag (persist the dismissal) + focus-on-dismiss, the conditional-live-region trap (shared with Toast), the CLS/layout-shift consideration, and the **Banner-vs-Alert-vs-`role=alert`** naming overload (the practice picks `Banner`). *stable* ‚Äî the twenty-sixth brief; the documented home for the errors and actions Toast hands off.
- **[Progress](progress.md)** ‚Äî the fourth Feedback brief: the loading/in-progress indicator, reusing Toast's live-region pattern for milestone/completion announcements. The determinate-vs-indeterminate fork (the `role=progressbar` value contract + the **don't-fake-determinate** rule; indeterminate omits `aria-valuenow`, sets `aria-busy`), the live-region milestone announcement (**don't-announce-every-percent** ‚Äî throttle to 25/50/75/100), the **progress-vs-spinner-vs-skeleton** three-loading-patterns decision and the **`meter`-vs-`progressbar`** semantic boundary (meter = static measurement, a separate component), linear/circular/dashboard/global placement + Material 3's **Stop Indicator** (the 4dp terminal dot for WCAG 1.4.11), the **anti-flash** delay/min-display rules (~200‚Äì500ms / ~500ms), and the reduce-motion-can't-remove-the-only-indicator substitution. *stable* ‚Äî the twenty-eighth brief; only Empty State remains in Feedback.
- **[Empty State](empty-state.md)** ‚Äî the fifth and final Feedback brief, which **completes the Feedback category**: the rendered content of a data container when there's nothing to show, turning absence into guidance. The **genre taxonomy** (first-use/no-results/all-done/error/no-permission) that drives content + action, the loading/content/empty/error **terminal-states machine**, the genre‚Üíaction mapping, the illustration question (decorative/`aria-hidden`, the over-illustration + dark-mode `currentColor` cost), the **static-content-is-not-a-live-region vs the dynamic no-results `role=status` announcement** (the lightest reuse of the Feedback live-region pattern) + **focus-stays-in-search**, the **table-replacement methodology** (full-shell for first-use, preserve-headers for no-results), the **FOES** trap (`if (!isLoading && data.length === 0)`), the `headingLevel` outline injector, and content/tone by genre (never-blame, never-leave-a-blank-container, first-use-as-onboarding). *stable* ‚Äî the twenty-ninth brief; composed by the Data hosts (Table/List/Tree) as their zero-state.
- **[Inline Message](inline-message.md)** ‚Äî the third Feedback brief and the catalogue's **first explicit cross-category interlock**: the field/section contextual message that closes the live-region trio (Toast established, Banner refined, Inline Message applies to field association) and stitches Feedback to Form ‚Äî the field-level case *is* Text Field's helper/error slot, the primitive form controls compose. The field-vs-section scope, the **settled announcement model** (`aria-describedby` + `aria-invalid` + focus management, **not** a field-level live region ‚Äî the double-announce bug), reward-early/punish-late validation timing, the GOV.UK error-summary + visually-hidden "Error:" prefix + label-parity patterns, the helper‚Üîerror slot swap, the static-vs-dynamic refinement, severity + color-not-sole-carrier, and the reserve-space-vs-dynamic-expansion layout-shift model (+ the `position:absolute` anti-pattern). *stable* ‚Äî the twenty-seventh brief; closes the live-region trio.

### Overlay

- **[Menu](menu.md)** ‚Äî a transient overlay of application *commands*, and the catalogue's sharpest semantic boundary: the menu-vs-listbox-vs-navigation disambiguation (the "do / go / choose" test ‚Äî `role=menu` is for actions, *never* a dropdown of nav links, *never* value-selection; the most-misused ARIA pattern on the web), the roving-tabindex focus model (real DOM focus moves *into* the menu ‚Äî the explicit opposite of Combobox's `aria-activedescendant`), the menu-button `aria-haspopup`/`expanded`/`controls` contract, the submenu safe-triangle, disabled-but-focusable items, stateful `menuitemcheckbox`/`menuitemradio`, and the dropdown/context-menu/menubar surfaces. *stable* ‚Äî the eighteenth brief; the first to inherit the overlay/positioning substrate (Button trigger + the machinery Select and Combobox established). Filed here, not under Navigation, precisely because its thesis is that a menu is not navigation.
- **Dialog / Modal** ‚Äî *not yet briefed.* `scope: focus-trap and return, inert scope, Escape and overlay-click behaviour, native <dialog>+showModal(), z-index and scroll-lock, the modal-vs-drawer decision.`
- **[Dialog / Modal](dialog.md)** ‚Äî THE focus-trap overlay primitive, and the foundational Overlay brief: the native `<dialog>`+`showModal()` platform-first decision (the top layer, `::backdrop`, Escape, background inertness ‚Äî and the four gaps it leaves: scroll-lock + scrollbar compensation, light-dismiss/backdrop-click, initial focus, animation), the focus-trap + initial-focus + focus-return contract (never auto-focus a destructive button; the lost-invoker `<body>`-drop edge case), `role=dialog` vs `alertdialog`, when to disable light-dismiss (data loss) and the native `closedby` attribute, scroll-lock + scrollbar-width compensation, and the `@starting-style`/`allow-discrete`/`overlay` exit-animation solution. *stable* ‚Äî the twentieth brief; the trap counterpart to Menu's roving tabindex and the modal counterpart to Popover's light-dismiss (next), and the platform-first precedent set by Accordion applied to overlays.
- **[Drawer / Sheet](drawer.md)** ‚Äî the edge-anchored Overlay that closes the Overlay arc: a modal drawer is structurally a Dialog anchored to a viewport edge (inheriting the focus-trap, scrim, top layer, scroll-lock, native `<dialog>`, `@starting-style` animation, close/footer), a non-modal/persistent drawer pushes/squeezes the page and stays interactive. The modal-vs-non-modal (overlay-vs-push) decision and the fractured field philosophy (Chakra discourages non-modal; Fluent/Carbon embrace the inline panel; Material's temporary/dismissible/permanent), edge anchoring via logical placement (RTL mirrors with zero JS), the bottom-sheet detents/snap-points + drag physics (vaul/iOS) with the mandatory button alternative (WCAG 2.5.1/2.5.7), the focus model split (modal traps; non-modal is a `complementary`/`navigation` landmark), the native-`<dialog>`-can't-do-non-modal DOM split, and the focus-loss/portal-loop trap (an inner Combobox/Select portal escaping the drawer's trap). *stable* ‚Äî the twenty-third brief; the fifth and final Overlay brief, **completing the Overlay category** (Menu, Dialog, Popover, Tooltip, Drawer). `scope: edge-anchored placement, the drawer-vs-modal-vs-popover decision, focus management, mobile vs. desktop.`
- **[Popover](popover.md)** ‚Äî the generic anchored transient surface, the third Overlay pole, and the pivotal brief that **formalises the positioning substrate** Select, Combobox, and Menu have referenced since Select: the native Popover API (`auto`/`manual`/`hint`) + CSS Anchor Positioning (`anchor()`/`position-area`/`position-try`/`anchor-size` ‚Äî displacing Floating UI) and its **Containing-Block Trap** (the IMCB that breaks positioning silently), the positioning model the whole family inherits, the light-dismiss + non-trapping focus contract (vs Dialog's trap; move focus in only for interactive content), the native-popover accessibility gap (`popovertarget` wires behaviour not semantics ‚Äî add `aria-expanded`/`aria-controls`/role), role-by-content (`dialog`/`listbox`/`menu`/`tooltip`), and the popover-vs-tooltip-vs-menu-vs-dialog boundaries (the rich-vs-plain interactive/hover line the sharpest). *stable* ‚Äî the twenty-first brief; the light-dismiss + anchoring counterpart to Dialog's modal trap, and the base Menu / the Select & Combobox listbox / Tooltip are all specialisations of. Closes the substrate loop the catalogue carried since Select.
- **[Tooltip](tooltip.md)** ‚Äî the cleanest close to the Overlay arc: a `role=tooltip` specialisation of Popover (the `popover="hint"` layer) that adds the WCAG 1.4.13 hover/focus contract (dismissable/hoverable/persistent), the hover-AND-focus trigger and the touch/no-hover reality (the never-essential-info rule), `role=tooltip` + `aria-describedby`-vs-`labelledby` and the trigger-must-be-focusable / disabled-control trap (`aria-disabled` + event suppression), the plain-text/no-interactive boundary with Popover (the rejected Material 3 "rich tooltip" ‚Äî it's a Popover), the tooltip-vs-toggletip distinction (hover-describes vs click-discloses-via-live-region, the touch-accessible sibling), the `title`-attribute anti-pattern, and the warm-up/cool-down delay timer (focus bypasses the warm-up). *stable* ‚Äî the twenty-second brief; the fourth Overlay brief, completing the content-role coverage of the overlay family (`menu`/`dialog`/generic/`tooltip`).
