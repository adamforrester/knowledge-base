**The 33 §3 "10% of first reseller's cost" payback figure is a practitioner-extension claim, not a directly-citable Spectrum/Carbon figure.** What Spectrum and Carbon *do* publish is the architectural pattern that *enables* per-brand programmatic generation:

Adobe Spectrum (per `spectrum.adobe.com/page/color-system/`):
- Color tokens use semantic naming with index notation: `gray-50`, `gray-75`, `gray-100`, ..., `blue-100`, `blue-1000`, etc.
- "Grays 50 through 200 are used for background layers" — semantic role mapping
- "When not using background layers, gray-100 is the default background color"
- Per `opensource.adobe.com/spectrum-web-components/tools/styles/`: "Raw color definitions — Color scale values that differ between light and dark themes: `--spectrum-gray-50` is white (255, 255, 255) in light theme, but dark in dark theme"
- Per `adobe.design/ideas/naming-colors-in-design-systems`: "In our dark color theme, `blue-100` is darker than `blue-400`. In all themes, `gray-50` is lighter than `gray-100`."

The Spectrum pattern verified: a *single color name* (`gray-50`) resolves to *different values per theme* via theme-swap. The semantic role (lighter vs. darker; default-background vs. layered-background) is preserved across themes.

IBM Carbon (per `carbondesignsystem.com/elements/themes/overview/`):
- "package per theme for white, g10, g90, and g100"
- "Each theme is assigned universal color variables, which are determined by common roles and usage"
- Per `v10.carbondesignsystem.com/guidelines/color/usage/`: "Tokens by theme. White G10 G90 G100. Core color tokens... `$ui-background` Default page background. White; —; #ffffff. ..."
- Per `react.carbondesignsystem.com/?path=/docs/components-theme--overview`: "g100. g90. g10. white" — four-theme switching
- Per `svelte.carbondesignsystem.com/llms-full.txt`: theme switching uses `document.documentElement.setAttribute` with theme name (`g10` | `g80` | `g90` | `g100` | `white`)

The Carbon pattern verified: four canonical themes (`white`, `g10`, `g90`, `g100`) shipped as separate CSS packages with a runtime theme-swap mechanism. Tokens like `$ui-background` resolve to per-theme values; the semantic role is preserved.

What this means for the 33 §3 "10%-of-first-reseller" payback figure:
- Neither Spectrum nor Carbon *publish* the 10% figure.
