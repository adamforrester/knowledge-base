WHAT THE TOC CONFIRMS

1. The acronym MINDE appears in the spec's section headings — both at the
   conceptual level (§14.1.2 "Closest Color (MINDE)") and at the algorithmic
   level (§14.2.1 "Binary Search Gamut Mapping with Local MINDE"). MINDE is
   plausibly "Minimum Delta-E" given the spec's use of ΔE2000 and ΔEOK as
   distance metrics in §20, but the expansion is not literally written out
   in any heading captured.

2. The CSS spec presents three named algorithms as siblings in §14.2:
   - Binary Search Gamut Mapping with Local MINDE
   - EdgeSeeker Gamut Mapping
   - Ray Trace Gamut Mapping
   Whether one is normatively designated as the CSS *default* could not be
   verified from the captured TOC — the body-text designation lives in
   §14.2's intro paragraphs, which were truncated.

3. The conceptual building blocks named in §14.1 ("Closest Color (MINDE)",
   "Chroma Reduction", "Chroma Reduction with Local Clipping") align with
   the practitioner shorthand "chroma reduction" used in 31's body text. The
   *category* of the default algorithm (chroma-reduction in OKLCh/Oklab
   space) is supportable from the TOC structure; the precise algorithm name
   ("MINDE Chroma Reduction with ΔE_ok distancing", as the external pass to
   31's run named it) needs a body-text capture before it can be quoted
   verbatim in a client artefact.
