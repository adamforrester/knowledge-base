## Research scope — cover all of the following. Use these as the section spine of the output.

1. **The decision-tree-to-generative spectrum.** Pull apart what "adaptive UI" actually means as architecture. Five rungs: (1) pure rule-based personalization (no LLM); (2) LLM picks from a catalog of pre-built variants; (3) LLM modifies or composes pre-built templates; (4) LLM emits structured data, deterministic renderer takes over; (5) LLM generates code directly. For each: control surface, QA-ability, latency, inference cost, production reality. Test the brief's hypothesis that production work concentrates in the middle (Rungs 3–4). Note that v0 is build-time, not runtime. Note Vercel's documented retreat from Rung 5 toward Rung 4 — that direction of travel is the single most important implementation fact.

2. **LLM integration patterns.** Tool calls as a UI primitive — relocate the design control surface explicitly: the model decides *which tool* and *with which arguments*; the rendering is deterministic code the developer wrote in advance. Structured outputs and the constrained-decoding mechanism underneath. RAG as context grounding (upstream; does not render anything). Multi-step and agentic flows. Streaming and progressive materialization (preview Section 5). The other SDKs and the emerging interop layer (MCP, AG-UI, AI SDK `message.parts`).

3. **Multimodal considerations.** Pointer and keyboard as the eroding default — the hover-dependency failure, WCAG 2.2 SC 2.5.8 as the legal floor. Touch and gesture — discoverability cost, hit-target guidance. Conversational and typed natural language — the blank-prompt problem, hybrid surfaces, Saarinen's rejection of chat as the whole interface. Voice — `gpt-realtime` and Gemini Live shipped reality vs. the Siri delay; turn-taking as the unsolved part. Vision and camera input — Apple Visual Intelligence and Project Astra; the grounding step. Cross-device handoff — three incompatible mechanisms (OS-level proximity continuity, cloud account sync, partial sync as in ChatGPT memory) and why generative UI state has no canonical transferable form. Ambient and proactive surfaces — the inbox-shaped container.

4. **Memory and context architecture.** Five places state can live (session-only, persistent profile/CDP, vector store, context window, hybrid). How shipped memory features actually work — ChatGPT's structured-dossier-in-system-prompt is not RAG; Claude's periodic distillation; Gemini's retrieval over Google account history. Why the architecture choice is a governance choice — GDPR right-to-erasure for vectors and weights, machine-unlearning's unsolved status, California CPPA's narrow ADMT scope (and the advertising exclusion), data residency premiums, the EU AI Act's interpretive question on whether content-marking attaches to AI-generated *interfaces* not just media.

5. **Streaming UI as a primitive.** What it is and how it works (token streaming + partial structured output, RSC + Suspense, optimistic rendering). The AI SDK 3.4 token-to-object-level shift and what that says about structure. Shipped patterns — Anthropic Artifacts in sandboxed iframes, Live Artifacts, ChatGPT Canvas's diff/version model, v0's stream-into-preview, GitHub Copilot ghost text, Linear's reasoning panel and Agent Interaction Guidelines. What it changes — latency tolerance, interaction expectations (every product has a Stop button), the visual language of "in progress." The deepest design consequence: the unit is no longer "the finished screen" but the screen's trajectory through partial states.

6. **Evaluation, QA, and governance.** Why traditional QA breaks. Anthropic's three-layer methodology (code-based / model-based / human graders) and golden-dataset incremental construction. The tooling landscape (OpenAI Evals, Promptfoo, Braintrust, LangSmith, Langfuse, DeepEval, Ragas, Inspect) and the gap: none is purpose-built for generative UI. The weakest link — accessibility at scale, the irony that the foundation paper's strongest *evidence base* is what is hardest to QA in the generative case. Human-in-the-loop at thresholds; telemetry as the post-ship signal (Notion's 80% figure, regret signals); OpenTelemetry GenAI semantic conventions; red-teaming maturity (Microsoft's hundred-products study). Governance under-formalization — no design-system-specific outcomes-based framework yet.

7. **Cost, latency, and performance reality.** Per-million-token pricing across the frontier and mid-tier models as of May 2026. The compounding example: an identical 4k input / 1k output render at one million sessions/day swings ~12× across model tiers. Latency budgets vs. shipped first-token times. Caching realities — exact match (~18% hit), semantic (20–45% production), provider-side prompt caching as the most reliable lever (with the design discipline of stable-prefix prompt ordering). Routing and cascading; Notion's published routing approach. The strongest cost lever for adaptive UI: pre-compute and template-cache common layouts; reserve live generation for genuinely novel states.

8. **Authoring tooling and the design workflow.** The honest answer: there is no mature design tool for variable output. Figma + prompts (and the Figma Make limits — fixed prototype per prompt, manual edit breaks prompt-based editing). Storybook + eval suites as the most credible component-driven workflow. v0-style prompt-to-UI tools and the "70% problem." The unsolved problem is *designing for variability* — the academic vocabulary ("variability by design," architectural intent over visual authorship) is sharper than industry's. UICrit as the missing-review-primitive academic proposal. Component-prompt-pair as a directional idea without established literature.

9. **Design system implications.** Primitives over composed components — why shadcn/ui is "the default UI library of LLMs." Semantic component contracts as the highest-leverage design-system investment for adaptive UI. Evals replacing screen-by-screen QA. Governance shifting from compliance to outcomes — system prompt as versioned design artifact, variant logging as infrastructure. The synthesis: the design system moves *up* a level — from specifying pixels to specifying the space of acceptable outcomes — and that is more demanding work, not less.

10. **Where this might be headed (speculative).** Label everything in this section. The middle of the spectrum stays where the work is. An interop standard consolidates — but which one is genuinely open. The authoring-tool gap gets filled, or it doesn't and that is itself the answer. Falling token prices weaken the economic objection without removing it. Recommendation practice and generative practice converge at the layout level. Then state plainly what is hype.
