# Drawer (Sheet) — field-truth study (Claude pass)

## 1. Framing
A Drawer is an **edge-anchored overlay panel** — a surface that slides in from a viewport edge (left/right/top/bottom) to hold a focused task, a detail inspector, a filter set, or (on mobile) navigation. It closes the Overlay arc (Menu / Dialog / Popover / Tooltip / Drawer) and it **stands on Dialog**: a *modal* drawer is, structurally, a **Dialog anchored to an edge** — it inherits the focus-trap contract, the scrim, the top layer, scroll-lock, the native `<dialog>`+`showModal()` path, the `@starting-style`/`allow-discrete`/`overlay` exit animation, the close affordance, and the footer Buttons (see dialog). The Drawer is also the real form of the **mobile bottom-sheet** Dialog gestured at.

**The defining decision is modal vs. non-modal/persistent (overlay-vs-push):**
- **Modal drawer** — overlays the page with a scrim, **traps focus**, locks scroll (a Dialog at the edge): a focused task, or mobile navigation behind a hamburger.
- **Non-modal / persistent drawer** — **pushes or squeezes** the page content sideways, leaves the background **interactive**, does *not* trap focus or scrim: a desktop nav rail, a filter panel, a detail inspector you edit while referencing the page.

What it *isn't*: a **Dialog** (centered, no edge relationship — see dialog; the edge + the reference-while-working case are why you pick a drawer), a **Side Navigation** (a *persistent navigation structure*, not a transient panel — see side-navigation when briefed; a nav drawer is the *mobile/temporary* form of it), or a **Popover** (a small anchored transient surface — see popover). Why systems disagree: the modal-vs-non-modal taxonomy, the bottom-sheet detent/drag model, and the drawer/side-nav naming line.

## 2. Anatomy and parts
- **scrim / backdrop** — present for the **modal** drawer (inherited Dialog `::backdrop`); absent for the non-modal push variant.
- **edge-anchored surface** — the panel, pinned to an edge, full-height (side) or partial-height (bottom sheet); top layer when modal.
- **drag handle / grabber** — a small affordance at the dragging edge of a **bottom sheet** signalling swipe-to-dismiss/resize.
- **header** — title (`aria-labelledby`) + close button (icon Button, "Close").
- **body** — scrollable content (sticky header/footer; the body scrolls).
- **footer** — action Buttons (inherits Button), for task drawers.

## 3. Properties / API
- **`open` / `defaultOpen` + `onOpenChange`** — controlled/uncontrolled (inherited Dialog).
- **`placement` / `side`** — `left`/`right`/`top`/`bottom`, ideally via **logical** values (`inline-start`/`inline-end`/`block-start`/`block-end`) so RTL mirrors (§9).
- **`modal`** — modal (scrim + trap + scroll-lock — the Dialog path) vs non-modal/persistent (push, interactive background); default modal for temporary drawers.
- **`size` / `width`** — sm/md/lg or a value; full-bleed on small viewports.
- **dismissibility** — `closeOnEscape` + `closeOnOutsideClick` (modal); the persistent variant has its own toggle.
- **`snapPoints` / detents + `activeSnapPoint`** — the bottom-sheet partial/full-height stops (§ implementation).
- **drag/swipe** — enable swipe-to-dismiss/resize, **always with a button alternative** (§6).
- **focus** — `initialFocus`/`restoreFocus` (inherited Dialog); `trapFocus` true for modal, false for persistent.
- **role/aria** — `role=dialog`+`aria-modal` (modal) vs `complementary`/`navigation` landmark (non-modal — §6).

## 4. States and variants
- **States:** closed / open / **animating** (slide; the exit inherits Dialog's solution) / **dragging** + per-**snap/detent** (bottom sheet).
- **Variants:**
  - **modal drawer** (scrim, trap) vs **non-modal/persistent** (push, interactive). Material's taxonomy: **temporary** (modal, over content) / **dismissible** (slides away, can push) / **permanent/standard** (always present, part of layout).
  - **by content:** **navigation drawer** (the hamburger panel / nav rail) vs **utility/detail drawer** (filters, inspector, cart) vs **bottom sheet** (mobile, detents).
  - **placement** — inline-start / inline-end / block-start / block-end.
  - **size** — sm/md/lg/full.

## 5. Usage guidance
- **Use a modal Drawer** for a focused secondary task or a mobile navigation menu — when the user should deal with the panel before returning, and the edge anchoring (vs a centered Dialog) suits a tall list/form or the mobile idiom.
- **Use a non-modal / persistent Drawer** when the user needs to **reference or act on the main page while the panel is open** — a filter panel that updates results live, a detail inspector edited against the list behind it, a desktop nav rail. (This reference-while-working case is the main reason to choose a non-modal drawer over a modal Dialog — Carbon/Material's rule.)
- **Don't use a Drawer for:** content with **no edge relationship** that should sit centered (→ Dialog), a **persistent primary navigation structure** (→ Side Navigation — a drawer is its temporary/mobile form), or a **small contextual surface anchored to a control** (→ Popover/Menu).
- **Decision frameworks:**
  - **Modal vs. non-modal:** must the user finish with the panel first (modal, trap + scrim) or work the page alongside it (non-modal, push, no trap)?
  - **Drawer vs. Dialog:** edge-anchored + tall/list/nav/reference-while-working (Drawer) vs. centered + self-contained interruption (Dialog). On mobile, a Dialog often *becomes* a bottom-sheet Drawer.
  - **Drawer vs. Side Nav:** transient/collapsible panel (Drawer) vs. a persistent part of the page structure (Side Navigation); the mobile transform is side-nav → modal drawer behind a hamburger.
  - **Placement:** nav on the inline-start edge; detail/utility on the inline-end edge; the mobile sheet from the block-end (bottom).

## 6. Accessibility
- **Role by modality:** the **modal** drawer is a `role="dialog"` + `aria-modal="true"` (inherits Dialog's whole a11y contract — labelling via `aria-labelledby`, the focus trap, Escape, the inert background). The **non-modal/persistent** drawer is *not* a modal dialog — it's a **`complementary`** landmark (a detail/filter aside) or a **`navigation`** landmark (a nav drawer), labelled, and left in the normal page flow/tab order.
- **The focus model (the modality split):**
  - **Modal:** focus **moves in and is trapped**, Escape closes, focus **returns to the invoker** — all inherited from Dialog (initial focus never on a destructive button; the lost-invoker fallback).
  - **Non-modal/persistent:** focus is **not trapped** — opening it may move focus in (for a task) or not (for a persistent rail), and Tab exits to the page; it stays a normal landmark.
- **Scroll-lock** — present for modal (inherited Dialog + scrollbar compensation); absent for the non-modal push.
- **Gesture accessibility (the Drawer-specific a11y rule):** swipe/drag to open/close/resize a sheet must **never be the only way** — **WCAG 2.5.1 (Pointer Gestures)** requires a single-pointer alternative to any path-based gesture, and **2.5.7 (Dragging Movements)** requires a non-dragging alternative to any drag. So a bottom sheet that's swipe-to-dismiss must *also* have a close button (and tap-scrim), and detents reachable by gesture must have a non-drag control.
- **WCAG SCs:** 2.5.1 / 2.5.7 (the gesture rules — the headline addition), plus the modal drawer's inherited Dialog SCs (2.4.3 focus order/trap, 2.1.2 escapable, 4.1.2 name/role/value, 1.4.13/2.4.11 focus not obscured) and the landmark SCs (1.3.1, 2.4.1 bypass) for the non-modal case.

## 7. Content guidelines
- **Title** — names the panel's purpose ("Filters", "Order details"); the accessible name.
- **Body** — the task/reference content; keep a tall drawer scannable with a sticky header/footer.
- **Footer actions** — for task drawers (Apply/Cancel, Save); a filter drawer often applies live (no footer) — match the modal/non-modal intent.
- **Close affordance** — a labelled "Close" icon (top-trailing); a bottom sheet adds the **drag handle** as the visible resize/dismiss affordance (plus the button — §6).

## 8. Motion and transition
Slide from the anchored edge — a `translate` from off-screen + the Dialog substrate (`@starting-style`/`allow-discrete`/`overlay`, with the scrim fading for the modal variant). The non-modal push **animates the page margin/transform** alongside the drawer's slide. The **bottom sheet** tracks the drag in real time (1:1 finger-follow) and **springs to the nearest snap point** on release based on position + velocity (a physics/spring model, not a fixed duration — vaul is the reference). Under `prefers-reduced-motion: reduce`, drop the slide/translate to a fade or snap. (See dialog §8, 18-motion-foundations, 19-motion-implementation-web.)

## 9. Internationalization
- **RTL** — placement uses **logical** axes (`inline-start`/`inline-end`) so a "start-edge" nav drawer flips from left (LTR) to right (RTL) automatically; the slide direction and any chevrons mirror. The bottom sheet (block-end) is direction-neutral. Don't hardcode `left`/`right`.
- **Text expansion** — drawer content (forms, nav labels) expands; size to content with min/max bounds, fluid height; the bottom sheet's content height drives its detents. (See 13-internationalization-and-rtl.)

## 10. Naming
**`Drawer`** (Material, Ant, Atlassian, Chakra, Base Web, vaul) and **`Sheet`** (shadcn's Radix-Dialog edge panel; Apple's mobile term) are the two names; **`Panel`** / **`Side Panel`** (Fluent) is the non-modal/inline framing. **The practice default: `Drawer`** (the edge-anchored overlay, an edge-placement of Dialog) with **`placement`** + **`modal`** as props (not separate components), the bottom sheet as the `block-end`-placed mobile variant (with detents), and the persistent navigation case routed to **`Side Navigation`** (a drawer is its temporary/mobile form). Aliases to map: `Sheet`, `SidePanel`, `Panel`, `Tray`, `Flyout` (overloaded), `BottomSheet`.

## 11. Implementation notes
- **A modal drawer is a Dialog at the edge** — reuse the Dialog implementation (native `<dialog>`+`showModal()`, top layer, focus-trap, scroll-lock + scrollbar compensation, `@starting-style` animation, the close/footer); the *only* delta is the edge anchoring + the slide translate. Don't re-implement the modal machinery.
- **The non-modal/persistent layout** is a *different* mechanism — not an overlay but a **layout shift**: the page content gets a margin/`translate`/grid-column so the drawer **pushes/squeezes** rather than covering, the background stays interactive, no scrim, no scroll-lock, no trap. Decide overlay-vs-push at the modal boundary.
- **Bottom-sheet detents + drag physics** — `snapPoints` (e.g. `[0.4, 1]` of viewport height) with an `activeSnapPoint`; the surface **follows the finger 1:1** during drag and **springs to the nearest snap by position + velocity** on release; a downward fling past a threshold dismisses. **There is no native CSS detent/snap primitive** (scroll-snap is adjacent but not this) — it's JS (vaul, built on Radix Dialog, is the de facto reference; iOS sheets have native detents the web mimics). (Current-platform-state — verify.)
- **Gesture + button alternative** — implement swipe, but **always** ship a close button + tap-scrim + (for detents) a non-drag control (WCAG 2.5.1/2.5.7, §6); never gesture-only.
- **Don't hand-roll** — vaul (bottom sheet), Radix `Dialog`/shadcn `Sheet` (side drawer), React Aria, or Fluent `Drawer`; skin them.

## 12. Related and alternative components
- **Stands on:** Dialog (the modal-overlay substrate — focus-trap, scrim, top layer, scroll-lock, native `<dialog>`, `@starting-style` animation, close/footer; a modal drawer is a Dialog at the edge — see dialog), Button (footer/close — see button), Icon (close/handle — see icon).
- **Alternative to / boundary with:** Dialog (centered, no edge relationship — the edge + reference-while-working case picks Drawer), **Side Navigation** (a persistent nav structure; a drawer is its temporary/mobile form — see side-navigation when briefed), Popover (small anchored transient surface — see popover), the mobile bottom-sheet (which *is* a Drawer — the form Dialog's mobile transform takes).
- **Specialises into:** the navigation drawer, the utility/detail drawer, the bottom sheet.

## 13. Field POV evolution
1. **The native `<dialog>` made the modal drawer trivial** — a modal drawer is now `showModal()` + edge positioning + a slide, inheriting the top layer / focus / inertness rather than a bespoke portal+trap.
2. **vaul + iOS detents standardised the bottom sheet** — snap points, 1:1 drag tracking, and velocity-based spring-to-snap became the expected mobile pattern (a JS model; no native CSS detents yet).
3. **The modal-vs-non-modal clarity hardened** — systems separated the *overlay+trap* drawer from the *push+interactive* persistent drawer (and the persistent nav case migrated to Side Navigation).
4. **Gesture accessibility hardened** — WCAG 2.5.1/2.5.7 made the button-alternative-to-swipe non-negotiable.
5. **`@starting-style`/`overlay`** (inherited from Dialog) retired the JS animation library for the slide/exit (cross-brief platform-state — verify).
